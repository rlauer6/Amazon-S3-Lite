## Amazon::S3::Lite 1.0.1 - Bug Fixes & Build Refactor

### Summary

This release fixes two real S3 compatibility bugs that were masked
during development by LocalStack's more lenient SigV4 enforcement,
adds a `session_token` alias for STS credential compatibility, and
migrates the build system to `CPAN::Maker::Bootstrapper`.

### Bug Fixes

**Missing `x-amz-content-sha256` header on PUT requests**

Real S3 requires the `x-amz-content-sha256` header on all requests
with a body. `AWS::Signature4` includes the hash in the canonical
request but does not add the header explicitly - meaning it was absent
from the actual HTTP request sent to S3. `_request` now computes
`sha256_hex` of the content and sets the header before signing.
LocalStack accepts requests without this header, which is why this
was not caught in testing against the local stack.

**Session token not sent for STS/IAM credentials**

The token resolution in `_request` was constructing a coderef instead
of calling it, so `x-amz-security-token` was never set. Fixed by
resolving the method via `can()` and calling it correctly:

```perl
my $token_sub = $self->credentials->can('get_token') // $self->credentials->can('token');
my $token     = $token_sub ? $token_sub->($self->credentials) : undef;
```

**Logger stomping application log level**

`_init_logger` was unconditionally calling `Log::Log4perl->easy_init`
at WARN level, overriding any level set by the consuming application.
Now guarded with `Log::Log4perl->initialized` - if the application has
already configured Log4perl, the library leaves it alone.

**Credential object validation**

`_init_credentials` now uses `can()` to locate either the bare method
name or the `get_` prefixed variant, making it compatible with
`Amazon::Credentials` accessors (`get_aws_access_key_id`, etc.) as
well as plain accessors.

### New

- `Amazon::S3::Lite::Credentials` gains a `session_token` alias for
  `token` - improves compatibility with consumers that expect the
  standard STS naming convention
- `TESTING` section added to POD and README documenting LocalStack
  leniency caveats
- `t/00-amazon-s3-lite.t` - basic `use_ok` test
- `cpanfile` generated by `CPAN::Maker::Bootstrapper`
- `test-requires.skip` - prevents internal modules from appearing as
  external test dependencies

### Dependencies

- Added: `Digest::SHA` (core), `IO::Socket::SSL`, `Net::SSLeay`
- `JSON::PP` added to test-requires

### Build

Migrated to `CPAN::Maker::Bootstrapper` build system:

- `Makefile` fully replaced with bootstrapper-managed version
- New `.includes/`: `git.mk`, `help.mk`, `perl.mk`,
  `release-notes.mk`, `update.mk`, `upgrade.mk`, `version.mk`
- `docker-compose.yml` updated: added `logs`, `sts` to LocalStack
  services and `LOCALSTACK_AUTH_TOKEN` support
